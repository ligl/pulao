KeyZone（关键价格区域）是趋势、区间、反转等结构的集中点，反映市场历史供需力量。

 当价格接近或进入该区域时，市场必然在此处做出反应，因此识别价格行为对 KeyZone 的态度，是捕捉趋势延续、回调结束、假突破和真突破的核心。

为了可实现量化建模，我们将价格与 KeyZone 的关系梳理为 **四种本质性的作用方式**：

1. **Approach（接近）**

2. **Touch（触及）**

3. **Accept（接受）**

4. **Reject（拒绝）**

这四类构成了整个价格行为的最小完备系统，任何价格与 KeyZone 之间的相互作用都落入这四类中的一个。


---

# 1️⃣ **接近（Approach）**

### **定义**

价格尚未触及 KeyZone，但其运动方向、波动形态和速度特征表明「正在靠近该区域」。

### **市场含义**

接近阶段是趋势的延伸段，或反转的准备区域，是“下一步行为”的前置阶段。

### **可量化特征**

*  价格与 KeyZone 距离逐渐缩小（distance ↓） 

*  连续高点/低点朝 KeyZone 移动 

*  波动幅度出现压缩（ATR ↓） 

*  成交量整体减弱（吸引力集中） 

*  EMA 或趋势结构指向 KeyZone 

### **工程标签**

`approach = True/False`

 `distance_to_zone`（连续递减趋势）

 `approach_speed`（单位时间距离缩减量）


---

# 2️⃣ **触及（Touch）**

### **定义**

价格首次触碰 KeyZone 区域边界。

### **市场含义**

触及代表市场正式“询问”该区域的供需力量，但还未体现明确态度。

### **可量化特征**

*  影线或实体首次交叉 KeyZone 边界 

*  区域内部的成交尚少 

*  触及后常伴随波动放大或急促缩小 

### **工程标签**

`touch = True/False`

 `touch_time`（datetime）

 `first_touch_index`（索引）


---

# 3️⃣ **接受（Accept）**

### **定义**

价格进入 KeyZone 后在区域内部停留、吸收、震荡，表现为“接受该区域的供应/需求”。

换言之：区域内部出现持续成交 → 市场愿意在这里交易。

### **市场含义**

代表 KeyZone 的阻力/支撑正在被吸收，突破概率增大。

 趋势突破、反转确认、区间突破成功，都需要 accept 作为前置条件。

### **可量化特征**

满足任意条件即可：

*  多根 K 线实体落在区域内 

*  出现小级别震荡、inside bar cluster 

*  区域内成交量上升 

*  区域内部停留时间 ≥ 某阈值 

*  波动压缩（volatility contraction） 

*  价格反复进入与测试 

### **工程标签**

`accept = True/False`

 `accept_duration`

 `accept_strength`

 （如：在区域内的 K 线占比、成交量增加率）


---

### **补充：Accept 的穿透类型**

所谓“价格无视 KeyZone、直接穿透”，在量化定义上属于：

* **快速 Accept → Break**

Tick 层面必然存在区域内成交，因此不会构成单独类别。


---

# 4️⃣ **拒绝（Reject）**

### **定义**

价格触及 KeyZone 后迅速反向离开，显示区域供需仍然强大。

### **市场含义**

Reject 是趋势回调结束、区间反转、失败突破等模型的关键信号。

### **可量化特征**

*  触及后出现大实体反向 K 线 

*  长影线反转 

*  小周期快速攻击波段 

*  EMA、结构显示即时反转 

*  区域停留时间极短 

### **工程标签**

`reject = True/False`

 `reject_strength`（反转力度/速度）

 `reject_distance`（离开 KeyZone 的距离）


---

# 5️⃣ **为何四类关系构成完备系统？**

任何价格与 KeyZone 的相互作用从逻辑上必然属于以下之一：

1. **未触及 → Approach**

2. **触及但未表态 → Touch**

3. **进入并停留 → Accept**

4. **触及后离开 → Reject**

这是一个 **基于供需逻辑的有限状态机（FSM）**：

```plain

      +----------+
      | Approach |
      +----------+
            |
   price reaches zone
            v
      +----------+
      |  Touch   |
      +----------+
      /        \
Reject        Accept
  |              |
  v              v
离开          停留/穿越/突破
```
无论是趋势、震荡还是假突破，都能被完全覆盖。
不存在第五类，因为：

*  所谓“无视/直接穿透”在工程上属于 Accept 的快速模式 

*  市场逻辑上无法跳过区域 → 必出现内部成交 

因此四分类是 **最简完备系统**，适合量化工程实现。


---

# 6️⃣ **四类行为与趋势/区间结构的关系**

|价格行为|趋势回调模型|区间突破失败模型|
|:----|:----|:----|
|Approach|趋势延续段的回调靠近关键位|区间边缘的预突破准备|
|Touch|关键测试，准备反应|试探区间边界|
|Accept|回调结束→突破趋势线|真突破必要条件|
|Reject|回调失败→趋势恢复|假突破成立的关键行为|

量化实现时，可以基于每个 KeyZone 构建事件：

```plain

on_approach
on_touch
on_accept
on_reject
```
结合趋势结构（道氏）、波段结构（Swing）、EMA20/60，就能构建你的完整系统。

---

# 7️⃣ **工程化实现建议（Pulao 框架）**

1.  KeyZone 标注 

*  每个 KeyZone 存储 top/bottom、强度、类型 

*  建议使用 Polars 以窗口方式标注 

1.  价格状态机（FSM） 对每根 K 线判断价格与当前 KeyZone 的状态： 

```plain

state ∈ {approach, touch, accept, reject}
```
1.  量化因子 

*  approach_speed 

*  accept_duration 

*  accept_strength 

*  reject_strength 

*  touch_to_reject_delay 

*  touch_to_accept_delay 

*  zone_absorption_score（区域吸收度） 

1.  与交易模型联动 

*  趋势回调 → 接近 + 拒绝 

*  趋势突破 → 接受 → 上破 

*  假突破 → 接受失败 → 拒绝 

*  区间突破失败 → touch/reject 


---

# 🎯 **结语：KeyZone 的作用关系，是量化的最佳结构基石**

这四类价格行为描述了 **市场对供需墙的四种回答**，背后体现的是趋势动力、吸收过程、反转心理、突破机制。

你只需要在工程上建立：

*  KeyZone 标注系统 

*  价格行为状态机 

*  四类行为的量化指标 

你的“趋势回调突破模型”与“区间突破失败模型”都能自然融合。
